# Y笔记 Demo 需求说明文档

## 项目概述

本项目为 Y笔记的功能演示 Demo，主要展示两个核心功能模块的交互流程。所有数据均为 Mock 数据，重点在于展示完整的业务流程。

## 技术方案

- **前端框架**: React + TypeScript
- **样式方案**: TailwindCSS
- **构建工具**: Vite
- **数据来源**: 全部 Mock，无需后端
- **AI 效果**: 模拟延迟 + 预设的 Mock 结果

---

## 需求一：模板套用与格式整理功能

### 背景

针对销售公司提出的文件上传与引用需求，Y笔记需要支持用户完成文章写作后，按照预设模板自动整理格式并进行审校。

### 功能说明

#### 流程1：文档格式化

```
输入原文 → 选择模板 → 预览格式化结果（含错别字检查标注） → 提交
```

**功能点：**
- 用户输入/粘贴原文内容
- 选择预设的格式模板（类似 WPS 格式模板，包含标题字号、段落间距等样式）
- 系统自动按模板格式整理文档
- 错别字检查并高亮标注
- 预览格式化后的结果
- 确认无误后提交

#### 流程2：审批管理

```
配置审批流程（领导A → 领导B → 领导C） → 文档进入审批 → 逐级审批
```

**功能点：**
- 审批流程配置界面，支持设置审批人顺序
- 领导排位设置为可配置项（因领导排位每年可能变化）
- 文档提交后进入审批流程
- 展示当前审批状态
- 模拟逐级审批操作

### 页面规划

| 页面 | 功能 |
|------|------|
| 文档编辑页 | 输入原文、选择模板、预览结果、错别字高亮显示 |
| 审批配置页 | 添加/调整审批人，设置审批顺序 |
| 审批流程页 | 展示当前文档审批状态，模拟审批操作 |

## 需求二：周报月报合并与提炼功能

### 背景

针对销售公司各部门每月需要撰写日报、周报、月报的需求，实现从日报自动生成周报、从周报自动生成月报的智能合并功能。

### 核心逻辑

**智能合并规则：**
- 不是简单地将所有事项罗列相加
- 对延续性内容进行合并处理
- 自动计算整体进度（如某项工作本月完成了40%，不在月报中重复记录每周进度）

### 层级关系

```
每天写日报 → 选择时间范围生成周报 → 选择周报生成月报
```

### 功能说明

采用多 Tab 页结构：

| Tab | 功能 |
|-----|------|
| 日报 | 每天记录工作内容，列表展示历史日报 |
| 周报 | 日期范围选择器 → 从日报智能生成周报 |
| 月报 | 日期范围选择器 → 从周报智能生成月报 |

### 智能合并示例

**日报输入：**
```
- 1/1: 项目A完成10%
- 1/2: 项目A完成10%，项目B启动
- 1/3: 项目B完成20%
```

**周报输出：**
```
- 项目A：本周完成20%
- 项目B：本周启动并完成20%
```

### 页面功能

- **日报 Tab**:
  - 日报录入表单
  - 历史日报列表展示

- **周报 Tab**:
  - 日期范围选择器
  - 一键生成周报按钮
  - 周报预览与编辑
  - 历史周报列表

- **月报 Tab**:
  - 日期范围选择器
  - 一键生成月报按钮
  - 月报预览与编辑
  - 历史月报列表
